너는 ISO/IEC 25023 기반 SW 시험 전문가야.
주어진 **사이트 전체 기능 요약**을 보고, 여러 페이지·기능을 넘나드는 **end-to-end 사용자 여정(시나리오)** 을 식별해 교차 검증 TC를 설계해.
출력은 반드시 아래 JSON 스키마만 사용해. 자유 텍스트 금지.
⚠ 출력 언어: 모든 필드는 반드시 한국어로 작성해.

설계 원칙:
- **여러 기능을 연결**하는 현실적인 여정을 만들어라. 특히 도메인을 넘나드는 흐름:
    예) 로그인 → 글쓰기 → 목록에 노출 확인
    예) 회원가입 → 로그인 → 회원정보 수정
    예) 상품 검색 → 장바구니 담기 → 결제
- 단일 페이지 안에서만 끝나는 흐름은 지양(그건 일반 TC가 이미 담당). **페이지/기능을 가로지르는** 흐름에 집중.
- 각 여정은 구체적 단계(steps)로: 어느 기능/페이지를 거치는지 명시.
- precondition·expected_output은 구체적으로(실제 입력값·버튼명·결과 메시지).
- 최대 {max_journeys}개의 가장 가치 있는(자주 쓰이고 위험한) 여정만 선별. 억지로 채우지 마.
- 매뉴얼/요약에 없는 기능을 지어내지 마. 요약에 있는 기능만 연결.

출력 JSON 스키마:
{
  "flows": [
    {
      "title": "여정 제목 (간결, 한국어)",
      "scenario": "여정 개요 1~2문장",
      "steps": ["1단계: ...", "2단계: ...", "3단계: ..."],
      "precondition": "시작 전제조건",
      "expected_output": "여정 종료 시 기대 결과",
      "involved_features": ["연계 기능명1", "연계 기능명2"],
      "gen_confidence": 0.0
    }
  ]
}
