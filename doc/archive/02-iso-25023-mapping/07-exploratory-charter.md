# 탐색적 시험 Charter

**적용 범위:** L3 영역 일부 + 모든 Layer를 통합한 *발견 중심* 시험.
**책임 주체:** 시험원 (charter-based, time-boxed)

> **탐색적 시험의 위치:** AWT가 만든 TC는 *사전에 계획된 시험*. 탐색적 시험은 *시험원이 제품과 상호작용하며 발견하는 시험*. 둘은 보완적이며, *AWT는 charter 후보를 제안*하되 시험 자체는 시험원 전담.

---

## 1. Charter란?

**Charter:** 탐색적 시험 세션의 *목적·범위·시간 한계*를 1~2문장으로 명시한 미션 선언.

예시:
> "결제 페이지에서 비정상 카드 정보·취소·새로고침 등 *예외 상황*을 60분 탐색해 결제 무결성 시험."

좋은 charter의 조건:
- *한 문장*으로 표현 가능
- 시간 한계 명시 (30~120분)
- 측정 가능한 종료 시점
- 너무 좁지도 너무 넓지도 않음

---

## 2. Charter 양식

```
[Charter ID]: EXP-2026-MM-DD-NN
[Charter 제목]: <한 줄>
[목적]: <왜 이 탐색을 하는가>
[범위]: <어디까지>
[제외]: <어떤 부분은 안 본다>
[시간 한계]: ___ 분 (권장: 60~90분)
[탐색자]: <reviewer_id>

[탐색 출발점]
- 진입 화면·기능
- 사용할 데이터·계정
- 사전 가정

[탐색 노트]  (시험원이 실시간 기록)
시각 ___ : <관찰 / 시도 / 결과>
시각 ___ : ...

[발견 결함]
- 결함 1: 심각도, 재현, 권고
- 결함 2: ...

[시간 종료 후]
- 탐색 완료 / 추가 시간 필요 (사유)
- 다음 charter 제안
```

---

## 3. AWT가 제안 가능한 Charter 후보

자동 시험 종료 후 AWT가 *탐색 가치 높은 영역*을 식별해 charter 후보를 제안:

| 트리거 | Charter 후보 |
|---|---|
| L1에서 FAIL 비율 ≥ 10% 기능 발견 | "해당 기능의 *FAIL 패턴*을 60분 탐색해 근본 원인 파악" |
| L2에서 시험원이 `rejected` 비율 높았던 영역 | "AWT가 *반복적으로 잘못 만든 영역*에 대해 90분 탐색해 *AI가 놓친 결함*을 직접 발굴" |
| 매뉴얼에 *모호한 거동* 명시된 기능 | "매뉴얼이 정확히 정의하지 않은 *경계 거동*을 60분 탐색" |
| Cross-feature 시나리오 부족 영역 | "기능 A + B + C 결합 시나리오를 90분 탐색" |
| 결함 RAG에서 *유사 제품에 자주 발생한 결함 영역* | "과거 유사 제품에서 X 결함 다발 → 본 제품에서 동일 영역 60분 탐색" |

AWT는 이를 `output/Layer3_Aids/exploratory_charters_suggested.md`로 제공.

## 4. 세션 운영 가이드

### 4.1. 단일 세션 권장

- **길이:** 60~90분 (45분 후 짧은 휴식 권장)
- **목적 단일:** 하나의 charter만
- **노트 동시 기록:** 발견을 *놓치지 않는 가장 큰 장치*

### 4.2. 세션 도중 *목적 이탈* 처리

탐색 중 *charter와 무관한 결함 발견* 시:
- 즉시 멈추지 말고 *짧은 메모만* 기록
- 현재 charter 종료 후 별도 charter 생성해 후속 탐색

### 4.3. 세션 종료 후

- 노트 정리 → 결함 리포트로 분리
- 다음 charter 1~3개 제안 (이번 탐색에서 떠오른 의문 기반)

---

## 5. 시간 budget의 위치

L3 시간 분배 권장 (`04-human-only.md` §5 참조)에서 탐색적 시험은 **L3의 약 20~30%**:

```
L3 시간 (After AWT 가설, 시험원 1인당 제품 1개 기준)
├─ 사용성 평가 (heuristic + SUS)         : 30%
├─ 접근성 심층                           : 15%
├─ 부하·용량                              : 15%
├─ 보안 심층                              : 20%
├─ 탐색적 시험                            : 20%
```

조정 가능 — *제품 특성에 따라*.

---

## 6. 탐색적 시험 *기록의 자산화*

각 charter 세션의 노트는 *RAG의 가장 가치 큰 데이터*:
- AI가 만들지 못한 시나리오
- 도메인 직감으로 찾아낸 결함 패턴
- 매뉴얼 사각지대

이 노트들을 익명화 후(D16: L1+L2 자동, L3+L4 수동) RAG에 축적하면 *다음 제품 시험 시 AI가 유사 패턴 자동 제안* 가능 (Phase 2~3).

AWT는 노트 익명화 도구를 `tools/`에 제공 (구현 단계).

---

## 7. 사용자 표현 "*나머지 부분은 사람이 할 수 있다*"의 실체

탐색적 시험은 이 표현의 *가장 결정적* 영역.

- L1·L2는 AWT 강화로 *효율화*되는 영역
- L3 심층은 *전문가가 정해진 절차를 수행*하는 영역
- **탐색적 시험은 시험원이 *전문가로서 가치를 가장 크게 발휘*하는 영역**

AWT가 *L3 절차 영역의 시간을 줄여* 탐색적 시험에 더 많은 시간을 투자하게 하는 것이 본 프로젝트의 궁극적 가치 명제.
