# 품질특성별 결함 샘플

본 문서는 시험소가 과거 유사 제품에서 누적한 결함 패턴 예시. AWT가 신제품 TC 설계 시 참고.

## 기능적 적합성 (Functional Suitability)

### 패턴 F-1: 입력 길이 검증 누락
- 증상: 매뉴얼에 길이 제약이 있으나 실제로는 무제한 입력 가능
- 영향: 데이터베이스·UI 손상
- 시험 권장: boundary 기법 — 매뉴얼 명시 길이의 경계(포함/미포함) 양쪽

### 패턴 F-2: 권한 우회
- 증상: "본인만" 가능하다고 명시된 작업이 URL 직접 접근으로 가능
- 영향: 데이터 무결성·보안
- 시험 권장: negative_deep — 다른 사용자 계정으로 동일 자원 접근 시도

### 패턴 F-3: 페이지 경계 처리 결함
- 증상: 페이지네이션의 첫·마지막 페이지에서 화면 깨짐
- 영향: 사용성·기능 정확성
- 시험 권장: boundary — 페이지 1, 마지막 페이지, 마지막+1, 0, 음수

## 사용성 (Usability)

### 패턴 U-1: 에러 메시지 모호함
- 증상: "오류가 발생했습니다" 같은 일반 메시지
- 영향: 사용자가 무엇을 잘못했는지 모름
- 시험 권장: negative_deep — 잘못된 입력 시 메시지 4축 (위치·한국어·이해성·보안노출)

### 패턴 U-2: 영문 메시지 혼재
- 증상: 한국어 UI에 영문 에러 메시지
- 영향: 사용자 이해도 ↓
- 시험 권장: negative_basic 후 메시지 한국어 검증

## 신뢰성 (Reliability)

### 패턴 R-1: 동시 작업 시 데이터 불일치
- 증상: 여러 탭에서 동시 수정 시 한 쪽이 덮어쓰임
- 영향: 데이터 손실
- 시험 권장: state_transition — 두 세션에서 같은 자원 변경

## 보안 (Security)

### 패턴 S-1: XSS
- 증상: 입력값에 `<script>` 포함 시 실행
- 영향: 매우 큼
- 시험 권장: negative_basic — 스크립트 입력 후 다른 사용자에서 노출 확인

### 패턴 S-2: 인증 우회
- 증상: 로그인 없이 보호 페이지 접근 가능
- 영향: 매우 큼
- 시험 권장: negative_deep — 비로그인 직접 URL 접근
